# AI 

## 如何构建一个 AI 风格的应用？

我设想中，既然 LLM 发展这么迅速，后续我们再移动端可能就没有 apk 的概念了，全部由 AI 掌握，我们需要在框架内提供一些 Agent，这些 Agent 直接系统服务互动，没有了原始的 measure-layout-draw 流程。

所以就变成了，我们问 AI 帮我购物，AI 调用 Agent 获取供应商的数据，同时将结果使用我们这些绘制的 Agent 渲染成界面，在用户产生交互的时候，这个点击将会交给另外一个 Agent 来进行处理。

整体来看，实际上是从 “以应用为中心” 向 “以意图为中心” 的范式转移。

在这种模式下，开发者不再开发 App，而是开发 “数据能力插件” 和 “品牌风格特征包”（供 Agent 调用，确保买东西时看起来还是京东或亚马逊的调性）。

虽然愿景美好，但要实现 “无 measure-layout-draw”，需要解决两个关键点：

1. 语义对齐：如何确保 AI 描述的“购物卡片”在绘制 Agent 看来是美观且符合交互规范的？这可能需要一套语义化的图形协议。

2. 交互闭环：不仅仅是画出来，用户点击“购买”时的触摸事件，如何通过 SF 快速反馈给逻辑 Agent，而不经过复杂的窗口分发系统？
